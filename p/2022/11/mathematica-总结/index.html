Mathematica 总结

搞清楚 mathematica 与主流语言的区别，并总结其特征

2022-11-23

学完了 mma 的基本语法，来总结一下一些最显著的特征吧。

mma 难学的地方

函数式语言

mma 是一门函数式语言，不能用命令式语言的思想来使用 mma（我以前就是犯了这个错误导致根本不会用）。

然而很多人都没有函数式编程的经验，导致 mma 中的一些操作显得格外难以理解。

函数式编程的核心是函数以及它的输入输出，因此主要涵盖的内容就是数据结构、函数本身及模式匹配。

魔咒

mma 里面用到了特别多乱七八糟的符号：

@ // @@ # /@ _ -> & <||> :=

这些符号对于初学者来说完全不知所云，而且 mma 使用这些符号的频率非常高，导致入门的时候无从下手。就像 vim 初学的时候不会关闭编辑器一样，不学过 mma 都不知道怎么定义函数。

mma 中的常用符号也代表了不同的意思：

[] [[]] {}

前两个是数组、嵌套数组？第三个是块作用域？都不对！

下面主要聚焦于和普通语言的差别，因为内置的 8000 个函数可太多了。常用函数有时间再补上吧。

函数式语言

我们平常一般接触的 python、C 之类都是命令式语言，意思就是每一句话都在命令程序做一些东西。

但函数式语言不同，它的基本思想就是几个值输入到一个函数中，然后得到一个输出。著名的函数式编程语言有 Lisp、Haskell 和这里说的 mathematica 了。

要说有什么最显著的区别，可以看看 mathematica 是怎么处理条件的：

If[a > b, a, b] (*如果 a>b 就返回 a，否则返回 b*)

和命令式语言差别写法并不一致。只要理解了这一点，使用 mathematica 其实就并不困难了。

函数

调用函数

使用方括号调用函数（这一点和其他语言基本都不同）

RandomReal[10] (*这个函数的作用是返回一定范围内的*)

也可以使用别的符号调用函数：

f@g (*f[g]*)
g // f (*f[g]*)
f@g@x (*可以套娃*)

(*这里还牵涉到和mma底层储存相关的东西，有时间再写*)
f @@ {x,y,z} (*f[x,y,z]*)
f @@@ {{a, b, c}, {b, c, d}} (*{f[a, b, c], f[b, c, d]}*)

纯匿名函数（Pure Anonymous Function）

(*基本形式*)
f[#]&
(*例子*)
#^2 &
N[#] &
(*调用*)
f[#, a] & [a]
f[#, a] &@ a (*在只填入一个参数的情况下*)
f[#1, #2] & [a, b]
f[#a, #c] & [<|"a" -> 1, "c" -> 2|>] (*可以用字母填充，但需要使用 association*)
f[#["apples"], #["oranges"]] &[<|"apples" -> 10, "oranges" -> 12, "pears" -> 4|>] (*其实是这个的简写*)

Map

顾名思义，一个函数作用到 List 上。

Map[f, x]
f /@ {1,2,3} (*{f[1], f[2], f[3]}*)

mma 内置的数据结构有很多，比如数字、字符串，甚至颜色、图像。不得不承认，mma 在这方面上做得非常不错。

结构类型

mma 最值得学习的还是它的结构类型，这也是和普通语言最不同的地方。

(* List *)
{1,2,3}
{1,2,3}[[2]] (* 2 *)
{1,2,3}[[2;;3]] (* {2,3} 范围取值 *)
{{a, b, c}, {d, e, f}, {g, h, i}}[[1, 2]] (* b 多维数组取值 *)
{{a, b, c}, {d, e, f}, {g, h, i}}[[All, 1]] (* {a,d,g} *)
(* Rule *)
a -> 1
(* Association *)
<|a -> 1, b -> 2|>
<|a -> 1, b -> 2|>["a"] (* 1 *)

一言以蔽之，List 承担了包含数组、矩阵、元组等等的工作。取出里面的数要使用双括号，下标从一开始。

Association 类似于 python 字典 / js 对象 / C++ Map。使用单括号索引。

Rule 用于一些特殊的场合中，通常作为参数出现。

赋值

a = RandomReal[] (* 马上给 a 赋值 *)
b := RandomReal[] (* 等用到 b 再赋值，即延迟赋值*)
{b, b, b} (*{7.75203, 9.77809, 4.27623}*)

a -> 1 (*立即作用的 Rule*)
a :> 2 (*延迟作用的 Rule*)

= 是马上赋值， := 是延迟赋值。

模式匹配

模式匹配在函数式编程语言中也非常常用，下面快速了解一些模式匹配的魔咒：

Cases[{{a, a}, {b, a}, {a, b, c}, {b, b}, {c, a}, {b, b, b}}, {_, _}] (*_ 代表匹配任意字符*)
Cases[{{a, a}, {b, a}, {a, b, c}, {b, b}, {c, a}, {b, b, b}}, {b, _}]

(*__ 代表匹配任意结构，即有一个或多个元素*)
Cases[{{a, a}, {b, a}, {a, b, c}, {b, b}, {c, a}, {b, b, b}}, {__, b}] 

Cases[{{a, a}, {b, a}, {a, b, c}, {b, b}, {c, a}, {b, b, b}}, {a | b, _}] (* | 代表或*)

Cases[{f[1], g[2, 3], {a, b, c}, f[x], f[x, x]}, f[_]] (*可以匹配函数参数*)

(* /. 运算符，将其中的元素作用到模式匹配上*)
{{1, a}, {1, b}, {1, a, b}, {2, b, c}, {2, b}} /. {1, _} -> Red 

(* 命名函数匹配，意味着要相同的元素才能通过匹配 *)
Cases[{{a, a, a}, {a, a}, {a, b}, {a, c}, {b, a}, {b, b}, {c}, {a}, {b}}, {x_, x_}] 

Cases[{x, y, 3, 4, z, 6, 7}, _Integer] (* 匹配类型，理论上其实是匹配 Head *)
Cases[{99, x, y, z, 101, 102}, n_Integer -> {n, n}] (* 综合 *)

声明函数

为什么在 mma 里面声明一个函数这么麻烦呢……因为模式匹配的缘故吧。

(*翻译一下就是，对函数中 n 这个值进行匹配，并放到函数中进行运算*)
pinks[n_] := Table[Pink, n] 

(* /; 表示复合的匹配。例如这里意思就是匹配 n，且 n 同时满足是整数和 >0 的条件*)
pdigitback[n_Integer /; n > 0] := Framed[Reverse[IntegerDigits[n]]] 

(* 用到这种指定条件的模式匹配，就可以用一个函数定义多种条件了，例如这里就是 > 输出红色，<= 输出绿色 *)
(* 这也是函数式编程语言的一个特点 *)
check[x_, y_] := Red /; x > y
check[x_, y_] := Green /; x <= y

(* ___（三个下划线）代表零个或多个元素 *)
blackwhite[{___, Black, m___, White, ___}] := {1, m, 2, m, 3, m, 4} 

(* Longest 函数代表匹配最长的序列 *)
blackwhitex[{___, Black, Longest[m___], White, ___}] := {1, m, 2, m, 3, m, 4} 

(* .. 代表重复前面的匹配多次，这里的意思就是匹配最长的多次出现的白色或黑色*)
bwcut[{a___, Longest[(Black | White) ..], b___}] := {{a}, Red, {b}}
