Github Token的配置方法

2022-10-29

虽然 github 关闭密码验证已经是很久之前的事情了：Git password authentication is shutting down | GitHub Changelog，但是作为一个懒人，我在昨天才开始配置 token……

简单记录一下更换 token 的步骤（因为确实很长，且分散在不同的文档中），以免以后忘记。

1 添加 SSH key

生成 SSH key：Generating a new SSH key and adding it to the ssh-agent - GitHub Docs

添加 SSH key 到账号中：Adding a new SSH key to your GitHub account - GitHub Docs

简单来说就是在本地生成一个 SSH key（相当于凭证），然后保存在 github 自己的账号上，这样本机就可以通过这个凭证进行操作了。

大概步骤：

1、本地生成一个以自己邮箱为 label 的 SSH key：

ssh-keygen -t ed25519 -C "your_email@example.com"

2、输入储存的位置（回车则存储在默认位置）

> Enter a file in which to save the key (/c/Users/YOU/.ssh/id_ALGORITHM):[Press enter]

输入 passphrase （相当于保护 SSH key 密码一样的东西），也可以不设置直接回车。

> Enter passphrase (empty for no passphrase): [Type a passphrase]
> Enter same passphrase again: [Type passphrase again]

3、查看生成的 SSH key

cat ~/.ssh/id_ed25519.pub

然后复制

4、Github 配置

Settings -> Access / SSH and GPG keys -> new SSH key -> 起个名字，粘贴刚刚赋值的 key -> Add SSH key

2 生成 token

Creating a personal access token - GitHub Docs

真正的凭证应该是 token，有了 token 才能在命令行中与 remote 服务器交流。

一共有两种 token：classic 和 fine-grained 。最大区别在于 classic 只能应用到所有仓库上，fine-grained 可以更加细致的分配到什么仓库、什么权限上。

配置位置：Settings -> Developer settings -> Personal access tokens

配置详细见文档，应该也不是很难操作。

3 remote 最速解决方法

Managing remote repositories - GitHub Docs

git push, pull 等等应该是最常用的操作。带有 token 的 push 我倒是没在 github docs 上看到有相关的写法，导致一直在用修改前的 origin。

现在不能直接 push 到 https://github.com/name/repository.git，而是要在 github.com 前面加上 token@：

git push https://<GITHUB_ACCESS_TOKEN>@github.com/<GITHUB_USERNAME>/<REPOSITORY_NAME>.git

有可能需要修改一下以前的 remote 配置了：

git remote set-url origin https://<GITHUB_ACCESS_TOKEN>@github.com/<GITHUB_USERNAME>/<REPOSITORY_NAME>.git
