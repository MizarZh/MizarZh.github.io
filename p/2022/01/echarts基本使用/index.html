Echarts基本使用

2022-01-26

简介

本文仅是一个快速入手的基本使用简介，详细配置项请到官网去（真的非常详细！）

官网地址：https://echarts.apache.org/zh/index.html

因为过于复杂，通常需要一些资源的辅助以及持续的文档查询：

速查表：https://echarts.apache.org/zh/cheat-sheet.html

示例：https://echarts.apache.org/examples/zh/index.html

Ehcarts 是一个非常好用的数据可视化工具。

安装：

npm install echarts --save

js 中引入及初始化：

import * as echarts from 'echarts';
var myChart = echarts.init(document.getElementById('main')); // 取某个 dom 作为渲染
myChart.setOption({
  // 这一部分是 echarts 的核心，决定了数据和图标的类型
  title: {
    text: 'ECharts 入门示例'
  },
  tooltip: {},
  xAxis: {
    data: ['衬衫', '羊毛衫', '雪纺衫', '裤子', '高跟鞋', '袜子']
  },
  yAxis: {},
  series: [
    {
      name: '销量',
      type: 'bar',
      data: [5, 20, 36, 10, 10, 20]
    }
  ]
})

有时候需要按需引入：

// 引入 echarts 核心模块，核心模块提供了 echarts 使用必须要的接口。
import * as echarts from 'echarts/core';
// 引入柱状图图表，图表后缀都为 Chart
import { BarChart } from 'echarts/charts';
// 引入提示框，标题，直角坐标系，数据集，内置数据转换器组件，组件后缀都为 Component
import {
  TitleComponent,
  TooltipComponent,
  GridComponent,
  DatasetComponent,
  DatasetComponentOption,
  TransformComponent
} from 'echarts/components';
// 标签自动布局，全局过渡动画等特性
import { LabelLayout, UniversalTransition } from 'echarts/features';
// 引入 Canvas 渲染器，注意引入 CanvasRenderer 或者 SVGRenderer 是必须的一步
import { CanvasRenderer } from 'echarts/renderers';

// 注册必须的组件
echarts.use([
  TitleComponent,
  TooltipComponent,
  GridComponent,
  DatasetComponent,
  TransformComponent,
  BarChart,
  LabelLayout,
  UniversalTransition,
  CanvasRenderer
]);

且为了最大程度的减少使用的体积，可以使用构建工具选择仅需构建的模块。

组成

全局对象（echarts 本 chart）

引入的 import * as echarts from 'echarts' 就是这个全局对象。

主要用于创建和销毁 echarts 实例以及注册模块

echarts 实例（echartsInstance）

echarts 全局对象 init 后返回的实例就是 echarts 实例。

主要属性/方法：

1、setOption 方法：设置图表的数据和样式。

2、getWidth getHeight getDom getOption 等等 n 个读取配置的方法，以及改变大小的 resize 。

3、事件相关：触发事件 dispatchAction，绑定事件处理器 on，解绑 off。

4、showLoading, hideLoading

配置项

每次 echartsInstance 调用 setOption 函数时，就会更新图表样式。

是配置样式和数据的地方。基础配置见下。

配置项基础

将相应的配置写到配置表中相应的位置即可。

外围配件（非核心图表）

以及三维组件：

地球组件 globe，可用于地图散点图等的绘制中。

定义在如图所示的配置项中。

图表类型

以及三维图：

三维地图 geo3D、mapbox（三维散点图，飞线图，柱状图和地图综合） mapbox3D、三维笛卡尔坐标系图 grid3D。

配置在 series 中，通常格式：

[
    {
        type: line,
        id: 0,
        name: 'test1',
        ...
        data: [
            ...
        ]
    },
    {
        type: bar,
        id: 1,
        name: 'test2',
        ...
        data: [
            ...
        ]
    }
]

使用 type 来界定不同的图样，在 data 中指定该图的数据（数据在后面的数据有介绍），组件 id 用于 option 或 API 中引用组件，name 用于在 tooltip, legend 中的显示用。

有时候不同的图表可以使用不同的坐标系，如柱状图 bar 和折线图 line 都可以使用直角坐标系 grid，散点图 scatter 使用直角坐标系 grid 、极坐标系 polar 或地理坐标系 geo 都可以。

这些图表有一个设置为 coordinateSystem，用来设置使用什么坐标系。

但有些图比较特殊，有专门的坐标系，如地图 map、雷达图 radar等。这些仅有单一的坐标系，因此没有这个选项。

但是我们都能设置任何图表的轴 Index 的选项（如 grid 的 xAxisIndex, yAxisIndex 和 polar 的 polarIndex）来指定使用哪个轴，以达到组合图的效果。

轴

各配置的位置：

  • 直角坐标系轴配置：xAxis, yAxis

  • 极坐标系配置：radiusAxis, angleAxis

  • 地理坐标系：geo

  • 单轴/平行轴：singleAxis, parallelAxis

  • 雷达图坐标系：radar

  • 日历：calendar

以及三维图：

  • 三维地图坐标系： geo3D

  • mapbox：见 mapbox

  • 三维笛卡尔坐标系：xAxis3D, yAxis3D, zAxis3D

一个特殊的：

  • 直角坐标系内绘图网格，单个 grid 内最多可以放置上下两个 X 轴，左右两个 Y 轴。

可在轴上设定轴 Index，与图表相对应。

数据

series.data

通常为二维数组（bar, line 等），例如：

[
    // 前两列是第一、第二个坐标，用来标示位置（如xy值或极坐标的长度和极角）
    // 第三列和后续可能的列数是附加的第n维度，可用来区分颜色等。
    [1,2,3], 
    [2,3,4],
    ...
]

当然也有其他的 data 样式布局，详见各图表类型的配置文档。

dataset

用于单独的数据集声明，从而数据可以单独管理，被多个组件复用，并且可以自由指定数据到视觉的映射。

总之可以理解为分离出了一个单独存储数据的地方，且可以复用。映射有点复杂，详见配置文档。

特殊的数据引入

使用 echarts.registerMap 注册 geoJSON 或 svg 地理图，然后再在 map 图表类型配置中使用 {map: xxx} 选项引入。

其他

aria：无障碍

media：自适应

animation 一系列相关的设定。

总结

有很多很多的细节操作没讲，但是大概是把主线捋了一下，至少以后再见到 Echarts 不会一脸懵逼不会用了。
