Sequelize基本使用-1

1、介绍

Sequelize 是一个基于 Promise 的 Node.js ORM (Object Relational Mapping, 对象关系映射) 工具。

可以对很多数据库操作，支持 Postgres, MySQL, MariaDB, SQLite 等等数据库。

安装：

npm install --save sequelize
# 同时安装你需要的数据库
$ npm install --save pg pg-hstore # Postgres
$ npm install --save mysql2
$ npm install --save mariadb
$ npm install --save sqlite3
$ npm install --save tedious # Microsoft SQL Server

PS：因为是基于 Promise，因此要经常记得 await/async。

2、数据库创建

const { Sequelize } = require('sequelize');
// 第一种创建方法（URI）：
const sequelize = new Sequelize('sqlite::memory:')
const sequelize = new Sequelize('postgres://user:pass@example.com:5432/dbname') 
// 第二种创建方法（分开参数，sqlite）
const sequelize = new Sequelize({
  dialect: 'sqlite',
  storage: 'path/to/database.sqlite'
});
// 第三种创建方法（分开参数，其他语言）
const sequelize = new Sequelize('database', 'username', 'password', {
  host: 'localhost',
  dialect: /* one of 'mysql' | 'mariadb' | 'postgres' | 'mssql' */
});

测试连接：

try {
  await sequelize.authenticate(); // authenticate方法
  console.log('Connection has been established successfully.');
} catch (error) {
  console.error('Unable to connect to the database:', error);
}

关闭连接：

会自动关闭连接，但也可以使用 sequelize.close() 关闭连接。

可以设置 logging 设定，选择不同的记录。

3、Model

创建 Model

model 是数据库的抽象。

定义 model 的方法：

const { Sequelize, DataTypes, Model } = require('sequelize');
// 建立数据库：
const sequelize = new Sequelize('sqlite::memory:');
// 定义 model 方式一：使用 sequelize.define
// 第一个参数：model名，
const User = sequelize.define('User', {
  // 各列的定义 
  firstName: {
    type: DataTypes.STRING,
    allowNull: false
  },
  lastName: {
    type: DataTypes.STRING
    // allowNull 默认为真
  }
}, {
  // Other model options go here
});
// 定义 model 方式二：使用继承
class User extends Model {
    // 这里可以填入要继承的 Model，或者为空
}

User.init({
  firstName: {
    type: DataTypes.STRING,
    allowNull: false
  },
  lastName: {
    type: DataTypes.STRING
  }
}, {
  sequelize,
  modelName: 'User'
});

选项配置

freezeTableName

在定义时确定的有一个 model 名（例如上文的 “User”），而这个 model 需要对应一个数据库表名。freezeTableName 定义了这样一个关系：

如果为 false （默认），则数据库中的表名会自动复数（如 “User” -> “Users”）。

如果为 true，则数据库中的表名就是 model 名（如 “User” -> “User”）。

tableName

当然，如果这样的映射很麻烦，那么也可以直接指定这个 model 对应的数据库表名。tableName 就是这样一个参数。

timestamps

timestamps: false
// or
timestamps: true, // 就可以对以下两个参数进行修改
createdAt: false, 
updatedAt: 'updateTimestamp'

Model 同步

程序中定义的 Model 和数据库中的表不相同，甚至数据库中的表不存在，这种情况下我们就需要程序中的 Model 和数据库中的表相匹配，这时候就要用到 Model.sync() 方法。这个方法有三种用法：

await User.sync() // 会创建不存在的表格
await User.sync({ force: true }) // 如果同名表格存在，会先删除它，再创建新的表格
await User.sync({ alter: true }) // 会检测同名表格的定义，然后会对它进行必要的改变以适应 Model
await sequelize.sync() // 全部数据库进行同步

删除表格

await User.drop()
await sequelize.drop()

如果在 .sync(),.drop() 的选项中加入匹配 { match: /some RegExp/}，那么就能指定同步或删除某些符合匹配的 Model。

4、Model 的类型

各种类型

// String
DataTypes.STRING             // VARCHAR(255)
DataTypes.STRING(1234)       // VARCHAR(1234)
DataTypes.STRING.BINARY      // VARCHAR BINARY
DataTypes.TEXT               // TEXT
DataTypes.TEXT('tiny')       // TINYTEXT
DataTypes.CITEXT             // CITEXT          PostgreSQL and SQLite only.
DataTypes.TSVECTOR           // TSVECTOR        PostgreSQL only.

// Bool
DataTypes.BOOLEAN            // TINYINT(1)

// Int
DataTypes.INTEGER            // INTEGER
DataTypes.BIGINT             // BIGINT
DataTypes.BIGINT(11)         // BIGINT(11)

// Float
DataTypes.FLOAT              // FLOAT
DataTypes.FLOAT(11)          // FLOAT(11)
DataTypes.FLOAT(11, 10)      // FLOAT(11,10)

// Real
DataTypes.REAL               // REAL            PostgreSQL only.
DataTypes.REAL(11)           // REAL(11)        PostgreSQL only.
DataTypes.REAL(11, 12)       // REAL(11,12)     PostgreSQL only.

// Double
DataTypes.DOUBLE             // DOUBLE
DataTypes.DOUBLE(11)         // DOUBLE(11)
DataTypes.DOUBLE(11, 10)     // DOUBLE(11,10)

// Decimal
DataTypes.DECIMAL            // DECIMAL
DataTypes.DECIMAL(10, 2)     // DECIMAL(10,2)

// Unsigned & Zerofill integers - MySQL/MariaDB only
DataTypes.INTEGER.UNSIGNED
DataTypes.INTEGER.ZEROFILL
DataTypes.INTEGER.UNSIGNED.ZEROFILL

// Date 
DataTypes.DATE       // DATETIME: mysql / sqlite, TIMESTAMP WITH TIME ZONE: postgres
DataTypes.DATE(6)    // DATETIME(6) for mysql 5.6.4+. Fractional seconds support with up to 6 digits of precision
DataTypes.DATEONLY   // DATE without time

// UUID
DataTypes.UUID  // UUID: PostgreSQL and SQLite, CHAR(36): MySQL

更多支持的类型见：https://sequelize.org/v7/manual/other-data-types.html

每一列的设置

格式：

id: {
    type: DataTypes.INTEGER,
    primaryKey: true, // 主键
    autoIncrement: true, // 自增
    
},
name: {
    type: DataTypes.STRING,
    // 其他选项
    defaultValue: 'test', // 默认值
    allowNull: false, // 是否为空
	unique: true, // 是否为 unique，也可设置叫名称的约束
},
fieldWithUnderscores: { 
    type: DataTypes.STRING, 
    field: 'field_with_underscores' , // field: 叫另外一个名字
    comment: 'This is a column name that has a comment'
}, 

// 如果只有类型的话可以简写：
name: DataTypes.STRING
