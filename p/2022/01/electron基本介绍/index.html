Electron基本介绍

2022-01-28

简介

electron 是一个使用 JavaScript、HTML 和 CSS 构建桌面应用程序的框架。 嵌入 Chromium 和 Node.js 到 二进制的 Electron 允许您保持一个 JavaScript 代码代码库并创建 在Windows上运行的跨平台应用 macOS和Linux——不需要本地开发经验。

基本依赖：node, npm

# 创建程序
mkdir my-electron-app && cd my-electron-app
npm init # 在这里的 entry point 应为 main.js

# 安装 electron
npm install --save-dev electron

在 scripts 字段下增加：

{
  "scripts": {
    "start": "electron ."
  }
}

electron 的结构

electron 的结构分为三部分：main, preload, renderer

electron 由于使用了 Chromium，其内部构造也和浏览器很相似：

1、main 相当于浏览器本身，因此一个应用一次只能用一个 main 在运行。

它具有使用 require 模块和 nodejs API 的能力。

整个 electron 项目的入口也是它。

2、renderer 相当于各个页面，浏览器可以打开多个 tab 表名可以存在多个页面。

渲染器进程就像网页，无法直接使用 nodejs API。

入口是 index.html

3、preload 是在 renderer 加载前加载的进程，可以暴露部分 renderer 不允许使用的 API，或为 renderer 提前设置环境。

建议文件架构中就将这三个模块放到三个文件夹中，或者将 main 和 preload 将 renderer 分离开来。

接下来将讲解一些重要方法。

main

两个重要的模块：

app：控制应用程序的生命周期

BrowserWindow：创建和管理应用程序窗口。

app.on(...) // 在窗口在某些情况下会触发什么函数

// BrowserWindow 的用法
function createWindow () {
  const win = new BrowserWindow({
    width: 800,
    height: 600
  })

  win.loadFile('index.html') // 加载渲染器进程
}

要加载 preload 脚本，要在 BrowserWindow 的 webPreferences 选项中加上一项：

function createWindow () {
  const win = new BrowserWindow({
    width: 800,
    height: 600,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js')
    }
  })

主进程有各种各样的配置，建议查看官方的模板或 github 上适应各种情况的模板。

preload

preload 相当于 main 和 renderer 之间的桥梁。

设置事件：

window.addEventListener('DOMContentLoaded',  () => {
    ...
})

通信：

const { contextBridge } = require('electron')
contextBridge.exposeInMainWorld('myAPI', {
  desktop: true
})

// renderer.js
console.log(window.myAPI) // {desktop: true}

renderer

在 index.html 中加上：

<script src="./renderer.js"></script>

即算加入了渲染器进程（就把他们当作普通的网页看待就好）

渲染进程中的 ipcRenderer 方法可以用于进程间的事件通信。

// 渲染进程
ipcRenderer.invoke('some-name', someArgument).then((result) => {
  // ...
})

// 主进程
ipcMain.handle('some-name', async (event, someArgument) => {
  const result = await doSomeWork(someArgument)
  return result
})
