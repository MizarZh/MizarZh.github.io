Pm2介绍及使用

2022-01-25

简介

因为某项目使用 nohup 在后台挂机总是莫名其妙就断掉了很生气，因此我当场换成 pm2 帮我管理这些 node 项目。

pm2 是一个 node 进程管理工具，可以简化很多 node 应用管理的任务，如性能监控、自动重启

负载均衡等。

安装 pm2：

npm install -g pm2

核心命令

一些常见操作

启动、重新加载、重启、关闭、删除：

可以指定多个 app：

pm2 start app
pm2 reload app
pm2 restart app
pm2 stop app
pm2 delete app

app.js 也可以换成 all 代表全部，换成 id 代表这个进程的编号，换成字符串（如"npm run start"）代表执行 bash 命令。

启动可以给入的参数（start, restart）：

# 传入参数
-- arg1 arg2
# 更新环境（通常前面设置 NODE_ENV 改变）
--update-env

列出：

pm2 [list|ls|l|status]
# 后面加上参数，代表按照……进行排序
--sort [name|id|pid|memory|cpu|status|uptime][:asc|desc]

dashboard：

pm2 monit

显示元信息：

pm2 show app

重置重启次数：

pm2 reset app

pm2 也可以使用文件模式进行配置，因为太长本文不介绍。详情见 pm2 configureation file。

静态文件

pm2 serve <path> <port>

可以像普通的 app 一样指定参数。

SPA：

pm2 serve --spa

会自动定位到 index.html

基本 auth：

pm2 serve --basic-auth-username <username> --basic-auth-password <password>

重启策略

# 在某一个 cron 时间重启
--cron-restart="0 0 * * *"
--cron-restart 0 # 不使用 cron 重启

# 文件改变时重启
--watch

# 最大内存重启
--max-memory-restart 300M

# 重启延迟
--restart-delay=3000

# 不自动重启
--no-autorestart

# 
--stop-exit-codes 0

# 指数增长的重启延迟
--exp-backoff-restart-delay=100

记录（logs）

默认的记录文件在 $HOME/.pm2/ 文件夹中

显示记录

❯ pm2 logs -h

  Usage: logs [options] [id|name|namespace] # pm2 logs 选项 + app

  stream logs file. Default stream all logs

  Options:
    # 三种格式
    --json                json log output
    --format              formated log output
    --raw                 raw output
    # 只显示错误或标准输出
    --err                 only shows error output
    --out                 only shows standard output
    # 显示 n 行
    --lines <n>           output the last N lines, instead of the last 15 by default
    # 增加时间戳
    --timestamp [format]  add timestamps (default format YYYY-MM-DD-HH:mm:ss)
    # 仅打印 logs，不记录
    --nostream            print logs without lauching the log stream
    # 高光一些值
    --highlight [value]   highlights the given value
    -h, --help            output usage information

限制 log 大小，需要使用 pm2-logrotate 插件：

pm2 install pm2-logrotate

刷新 log：

pm2 flush [app]

开启 app 时指定的 log 设置

-l --log [path]              # 指定输出路径
-o --output <path>           # 指定标准输出路径
-e --error <path>            # 指定错误输出路径
--time                       # 使用标准时间戳记录
--log-date-format <format>   # 自定义时间戳记录
--merge-logs                 # 运行多个进程时使用同一个文件记录，不依照 id 区分

持续运行

$ pm2 startup
[PM2] You have to run this command as root. Execute the following command:
      sudo su -c "env PATH=$PATH:/home/unitech/.nvm/versions/node/v14.3/bin pm2 startup <distribution> -u <user> --hp <home-path>

复制并执行：

sudo su -c "env PATH=$PATH:/home/unitech/.nvm/versions/node/v14.3/bin pm2 startup <distribution> -u <user> --hp <home-path>

就可以了。

保存 app 列表，在重启时会马上再次运行：

pm2 save

回滚？（resurrect），回到 save 时的版本：

pm2 resurrect

取消开启时运行：

pm2 unstartup

cluster 集群

Node.js 的单个实例在单个线程中运行。 为了利用多核系统，用户有时会想要启动 Node.js 进程的集群来处理负载。集群模块可以轻松创建共享服务器端口的子进程。

pm2 中也可以使用 cluster 集群，传入 -i 选项：

pm2 start app.js -i max

max 代表 pm2 会自动探测有多少核的 CPU，并尝试运行最多的进程。
