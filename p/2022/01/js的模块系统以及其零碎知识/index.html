Js的模块系统以及其零碎知识

2022-01-28

原始状态

可以引入多个文件：

<script src="./index.js">
<script src="./app.js">

但是就算这么引入，也是全都在全局环境的：

function fn1() {
    ...
}
    
function fn2() {
    ...
}

会造成全局环境污染

使用对象

我们可以使用不同的对象写成模块：

var module = {
    fn1: function() {
        ...
    },
    fn2: function() {
        ...
    }
}
module.fn1()

但内部的私有变量会暴露。

IIFE

立即表达式

var module = (function() {
    var fn1 = function() {
        ...
    }
    var someValue = 1;
    return {
        fn1: fn1,
        someValue: someValue
    }
})();

这样内部的值就不会被修改。

AMD

define(id, dependencies, factory)
define('module', ['require', 'exports', 'other'], function(require, exports, other) {
   	require('other').someFunc()
    exports.fn1 = function() {
        ...
    }
})

已过时

UMD

一个通用化的模块系统。是 AMD 和 CJS 的结合体，为了解决这两种不同模块系统之间相互使用的问题。

(function (root, factory) {
    if (typeof define === 'function' && define.amd) {
        // AMD. Register as an anonymous module.
        define(['b'], factory);
    } else if (typeof module === 'object' && module.exports) {
        // Node. Does not work with strict CommonJS, but
        // only CommonJS-like environments that support module.exports,
        // like Node.
        module.exports = factory(require('b'));
    } else {
        // Browser globals (root is window)
        root.returnExports = factory(root.b);
    }
}(this, function (b) {
    //use b in some fashion.

    // Just return a value to define the module export.
    // This example returns an object, but the module
    // can return a function as the exported value.
    return {}
}));

CJS (CommonJS)

const someModule = require('someModule') // 全部导入
const { someValue } = require('someModule') // 解构，即部分导入

require('someModule') // promise

module.exports = someModule // 默认导出
module.exports.someValue = someValue // 导出 someValue 值

ESM (ECMAScript Module)

import someModule from 'someModule' // 全部导入
import { someValue } from 'someModule' // 解构，即部分导入

import('someModule') // promise

export default someValue // 默认导出
export someValue // 导出 someValue 值

可以在 html 中将 <script> 增加一个属性：<script type="module">，浏览器就能识别这是一个 es6 模块。

文件格式的问题

由于 CJS 在 node 上的广泛应用和 ESM 成为 ES6 标准的一部分，目前这两个模块系统都在使用。

有时候会使用文件名来区分这两种文件系统：

CJS: .cjs

ESM: .mjs

在文件识别时，会识别相应的内容。而对于没有区分的 .js 文件，则按照自己使用的模块系统来识别。

使用 cjs：
.cjs -> cjs
.js  -> cjs
.mjs -> esm
使用 esm:
.cjs -> cjs
.js  -> esm
.mjs -> esm
