Svg笔记

2022-02-16

简介

SVG 是一种 XML 语言，可以用来绘制矢量图形。

一些注意的点：

  • SVG的元素和属性必须按标准格式书写，因为XML是区分大小写的（这一点和HTML不同）
  • SVG里的属性值必须用引号引起来，就算是数值也必须这样做。

SVG 的种类：

目前最完整的版本是 1.1 版。目前SVG 2.0 仍在制定当中。

根元素和一点入门

SVG 是 XML 的一种方言，因此需要绑定命名空间（xmlns 属性）。

<svg version="1.1"
     baseProfile="full"
     width="300" height="200"
     xmlns="http://www.w3.org/2000/svg">
  <rect width="100%" height="100%" fill="red" />
  <circle cx="150" cy="100" r="80" fill="green" />
  <text x="150" y="125" font-size="60" text-anchor="middle" fill="white">SVG</text>
</svg>

rect, circle, text 元素就是用来绘制的，并可以在其上面添加属性以进行设置。

SVG文件全局有效的规则是“后来居上”，越后面的元素越可见。

坐标系统

原点在左上角，x 轴向右，y 轴向下。

SVG 可以设置画布大小，以进行缩放：

<svg width="200" height="200" viewBox="0 0 100 100"></svg>

这个语句说的是画布上可以显示的区域是 (0, 0) 到 (100, 100)。这个 100*100 的区域会被放大到 200*200 的画布上。

用户单位和屏幕单位的映射关系被称为用户坐标系统。除了缩放之外，坐标系统还可以旋转、倾斜、翻转。默认的用户坐标系统1用户像素等于设备上的1像素。

基本形状

矩形

<rect x="10" y="10" width="30" height="30"/>
<rect x="60" y="10" rx="10" ry="10" width="30" height="30"/>
<!-- x, y, width, height 不多解释，rx, ry 为圆角在x, y 方位的半径 -->

圆形

<circle cx="25" cy="75" r="20"/>

椭圆

<ellipse cx="75" cy="75" rx="20" ry="5"/> 
<!-- rx, ry 分别为椭圆的 x 半径和 y 半径 -->

线条

<line x1="10" x2="50" y1="110" y2="150"/>

折线

<polyline points="60 110, 65 120, 70 115, 75 130, 80 125, 85 140, 90 135, 95 150, 100 145"/>

多边形

<polygon points="50 160, 55 180, 70 180, 60 190, 65 205, 50 195, 35 205, 40 190, 30 180, 45 180"/>

路径绘制完后闭合图形。

路径

最强的一个元素，可以用来创建线条、曲线、弧形等。是使用指令式方法绘画的。

<path d="M 20 230 Q 40 205, 50 230 T 90230"/>

每一个命令都用一个关键字母来表示。大写表示绝对定位，小写表示相对定位。

直线命令

M

Move to（移动）命令。

语法：M x y 或者 m dx dy。作用是仅移动画笔，但不画线。

L

Line to（画线）命令。

语法：L x y。

H、V

horizontal, vertical。绘制水平线和垂直线

语法：H x 和 V y

Z

闭合路径命令。不需要参数，不区分大小写。

曲线命令

C

三次贝塞尔曲线。

语法：

 C x1 y1, x2 y2, x y (or c dx1 dy1, dx2 dy2, dx dy)

x, y 是曲线的重点，前两个点是控制点。

S

连续的贝塞尔曲线

语法：

S x2 y2, x y (or s dx2 dy2, dx dy)

如果 S 命令跟在一个 C 或 S 命令后面，则它的第一个控制点会被假设成前一个命令曲线的第二个控制点的中心对称点（反向）。

如果 S 命令单独使用，前面没有 C 或 S 命令，那当前点将作为第一个控制点。

Q

二次贝塞尔曲线

语法：

Q x1 y1, x y (or q dx1 dy1, dx dy)

T

二次贝塞尔曲线版本的 S

语法：

T x y (or t dx dy)

如果前面使用的不是 Q 命令或 T 命令，那么控制点被认为和终点是同一个点，那么画出来的就是一条直线。

弧形

语法：

A rx ry x-axis-rotation large-arc-flag sweep-flag x y
a rx ry x-axis-rotation large-arc-flag sweep-flag dx dy

rx, ry 指的是 x 轴半径和 y 轴半径。x-axis-rotation 是旋转角度。large-arc-flag 和 sweep-flag 分别用来描述不同情况下的弧。

最后两个参数指定弧形的终点。

填充与边框

SVG 可以使用多种方法着色：内联 CSS、内嵌 CSS、外部 CSS。

颜色

fill 属性设置对象内部的颜色，stroke 属性设置绘制对象的线条的颜色。可以使用颜色名（像red这种）、rgb值（像rgb(255,0,0)这种）、十六进制值、rgba值，等等。

fill-opacity 属性设置填充色的不透明度，stroke-opacity 同理。

描边

stroke-width属性定义了描边的宽度。

stroke-linecap属性控制边框终点的形状。可以使用 butt, squire, round 三个值。

stroke-linejoin属性用来控制两条描边线段之间用什么方式连接。

stroke-dasharray属性将虚线类型应用在描边上。

fill-rule，用于定义如何给图形重叠的区域上色；stroke-miterlimit，定义什么情况下绘制或不绘制边框连接的miter效果；还有stroke-dashoffset，定义虚线开始的位置。

CSS

CSS 同样也能使用这些值，只不过不应该使用 background-color 这类的值，应该使用 fill 这种 SVG 专有的属性。

渐变

必须给渐变内容指定一个id属性，否则文档内的其他元素就不能引用它。为了让渐变能重复使用，需要定义在 <defs> 标签内部。

线性渐变

<defs>
    <linearGradient id="Gradient2" x1="0" x2="0" y1="0" y2="1">
        <stop offset="0%" stop-color="red"/>
        <stop offset="50%" stop-color="black" stop-opacity="0"/>
        <stop offset="100%" stop-color="blue"/>
    </linearGradient>
</defs>
<rect fill="url(#Gredient2)"> <!-- 这样调用 -->

<linearGradient> 内部的 offset 属性表示的是偏移量，stop-color 表示在这一点上的颜色，stop-opacity 表示半透明度。

调用方法是在目标元素的 fill 中写上 url(#Gradient)。

还可以添加渐变方向，由 <linearGradient> 上的 x1, y1, x2, y2 确定。渐变色默认是水平方向的。

径向渐变

<defs>
      <radialGradient id="RadialGradient2" cx="0.25" cy="0.25" r="0.25">
        <stop offset="0%" stop-color="red"/>
        <stop offset="100%" stop-color="blue"/>
      </radialGradient>
</defs>

第二个点被称为焦点，由fx和fy属性定义。第一个点描述了渐变边缘位置，焦点则描述了渐变的中心。

<defs>
    <radialGradient id="Gradient"
          cx="0.5" cy="0.5" r="0.5" fx="0.25" fy="0.25">
        <stop offset="0%" stop-color="red"/>
        <stop offset="100%" stop-color="blue"/>
    </radialGradient>
</defs>

额外的一个属性：spreadMethod，该属性控制了当渐变到达终点的行为。可以有三个值：pad reflect repeat。

两种渐变都有一个叫做gradientUnits（渐变单元）的属性，它描述了用来描述渐变的大小和方向的单元系统。该属性有两个值：userSpaceOnUse 、objectBoundingBox。默认值为 objectBoundingBox。

图案

需要放在 <defs> 内部。

<defs>
  <pattern id="Pattern" x="0" y="0" width=".25" height=".25">
    <rect x="0" y="0" width="50" height="50" fill="skyblue"/>
    <rect x="0" y="0" width="25" height="25" fill="something"/>
    <circle cx="25" cy="25" r="20" fill="something" fill-opacity="0.5"/>
  </pattern>
</defs>
<rect fill="url(#Pattern)" stroke="black" x="0" y="0" width="200" height="200"/>

在 <pattern> 内部可以包含任何之前包含过的其它基本形状，并且每个形状都可以使用之前学习过的任何样式样式化，包括渐变和半透明。定义的是一个单元系统以及他们的大小。

类似于 gradientUnits，parttern 也有一个属性 patternUnits。

pattern 还有另外一个属性 patternContentUnits，它描述了pattern元素基于基本形状使用的单元系统，这个属性默认值为userSpaceOnUse。

文字

基本属性

<text x="10" y="10">Hello World!</text>

text-anchor：start, middle, end, inherit，允许决定从这一点开始的文本流的方向。

fill, stroke：用来上色。

字体属性

font-family、font-style、font-weight、font-variant、font-stretch、font-size、font-size-adjust、kerning、letter-spacing、word-spacing和text-decoration。

其他属性

tspan

该元素用来标记大块文本的子部分

<text>
  xxx
  <tspan font-weight="bold" fill="red">This is bold and red</tspan>
  xxx
</text>

属性：

x：为容器设置一个新绝对x坐标。它覆盖了默认的当前的文本位置。这个属性可以包含一个数列，它们将一个一个地应用到tspan元素内的每一个字符上。

dx：从当前位置，用一个水平偏移开始绘制文本。这里，你可以提供一个值数列，可以应用到连续的字体，因此每次累积一个偏移。

此外还有属性 y 和属性 dy 作垂直转换。

rotate：把所有的字符旋转一个角度。如果是一个数列，则使每个字符旋转分别旋转到那个值，剩下的字符根据最后一个值旋转。

textLength：给出字符串的计算长度。它意味着如果它自己的度量文字和长度不满足这个提供的值，则允许渲染引擎精细调整字型的位置。

tref

允许引用已经定义的文本，高效地把它复制到当前位置。并且可以独立地修改它的样式。

<text id="example">This is an example text.</text>

<text>
    <tref xlink:href="#example" />
</text>

textPath

该元素利用它的 xlink:href 属性取得一个任意路径，把字符对齐到路径，于是字体会环绕路径、顺着路径走。

<path id="my_path" d="M 20,20 C 40,40 80,40 100,20" fill="transparent" />
<text>
  <textPath xmlns:xlink="http://www.w3.org/1999/xlink" xlink:href="#my_path">
    This text follows a curve.
  </textPath>
</text>

变换

使用 transform 属性。

平移：translate(x, y)

旋转：rotate(angle)

斜切：skewX(val), skewY(val)

缩放：scale(x[, y])，如果 y 不提供，则假定 y = x

变形矩阵：matrix(a, b, c, d, e, f)，相当于矩阵： $$ \begin{bmatrix} a && c && e \newline b && d && f \newline 0 && 0 && 1 \end{bmatrix} $$ 需要注意的是，如果使用了变形，那么在元素内部就会建立一个新的坐标系统。

同样的，可以在 SVG 里面嵌入 SVG，可以利用 SVG 的 viewBox 等属性创建一个新的坐标系统。

剪切和遮罩

剪切用来移除在别处定义的元素的部分内容。在这里，任何半透明效果都是不行的。它只能要么显示要么不显示。

遮罩允许使用透明度和灰度值遮罩计算得的软边缘。

剪切

同样，定义在 <defs> 中，并使用 clip-path 引用

<defs>
  <clipPath id="cut-off-bottom">
    <rect x="0" y="0" width="200" height="100" />
  </clipPath>
</defs>
<circle cx="100" cy="100" r="100" clip-path="url(#cut-off-bottom)" />

遮罩

<defs>
    <linearGradient id="Gradient">
      <stop offset="0" stop-color="white" stop-opacity="0" />
      <stop offset="1" stop-color="white" stop-opacity="1" />
    </linearGradient>
    <mask id="Mask">
      <rect x="0" y="0" width="200" height="200" fill="url(#Gradient)" />
    </mask>
    <rect x="0" y="0" width="200" height="200" fill="red" mask="url(#Mask)" />
</defs>

滤镜

元素：<filter>，定义在 <defs> 中。

内部可以添加不同的效果，如 <feGaussianBlur>, <feOffset> 等等。然后在应用的元素上设置属性 filter="url(#MyFilter)"。
