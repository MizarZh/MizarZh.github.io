Docker介绍

2022-02-25

来源：https://docs.docker.com/get-started/overview/

使用名词：

daemon - 守护进程

container - 容器

client - 客户端

image - 镜像

介绍

Docker 是一个用于开发、运输和运行应用的开源平台。Docker 可以让我们的应用与运行平台分离，以实现快速的软件分发。

Docker 平台

Docker 提供了一个较为独立的容器（container），container 有能力进行打包和运行应用。这种独立性和安全性允许我们可以在同一个主机上运行多个 container。 container 是轻量级的，但完整包含了运行应用所需的一切依赖，因此在主机上也无需安装响应依赖。你可以轻松的分享工作时的 container ，它们在不同的机器上运行的方法是一样的。

Docker 架构

Docker 使用了 C-S（Client-Server）架构。Docker client 与 Docker 的守护进程（daemon）交流，用来运行命令。Docker 的 client 和 daemon 可以运行在同一个系统上，或者 client 连接一个远程的 daemon。Docker 的 client 与 daemon 的交流通过 REST API，运行在 UNIX 套接字或其他网络界面上。另一个 Docker client叫做 Docker Compose，这个工具可以让我们在有一组容器上的应用工作。

Docker 的守护进程名称为 dockerd。

Docker 守护进程

用于听取 API 请求，以管理 Docker object，如 images，containers，网络和容量等。daemon 同样也能和其他 daemon 进行交流来管理 Docker 服务。

Docker 客户端

Docker Client 是用户与 Docker 交互的主要手段。

Docker Desktop

是一个在 Mac 和 Windows 上易于安装和使用的应用，包含了 Docker daemon 、Docker client、Docker Compose、K8s 等工具。

Docker registries

Docker registry 保存着 Docker images。Docker Hub 是一个公用的 registry，所有人都可以使用，Docker 默认都会在 Docker Hub 上查找 image。

当我们使用 docker pull 或者 docker run 命令时，所需的 image 就会被拉取到配置好的 registry 中。使用 docker push 命令时，image 就会被推送到 registry 中。

Docker Object

image

一个 image 是一个只读的、用于创建 container 的模板。通常一个 image 是基于另一个 image，在加上一些定制。我们可以直接拉取别人的 image，也可以创建自己的 image。

创建一个 image，需要创建一个 Dockerfile，定义需要使用什么步骤创建一个 image，以及如何运行它。在 Dockerfile 中的每一个命令都会在 image 上添加一层，如果修改了其中的一些命令，仅需修改相应的层就可以再运行了。这就是为什么 image 又轻又快的原因。

conatiner

container 是可运行的、image 的实例。我们可以创建、启动、停止、移动或删除一个 container。

container 与主机是相对独立的，但我们可以控制独立的程度。例如网络、储存。

Docker 常用命令

在容器内运行程序

docker run 命令可以在容器内运行一个应用程序。

例如：

docker run ubuntu:15.10 /bin/echo "Hello world"

ubuntu:15.10：即指定要运行的镜像。如果本地不存在，则在 Docker Hub 下载。

/bin/echo "Hello world"：在容器中运行的命令

在 docker 中运行交互式的容器：

docker run -i -t ubuntu:15.10 /bin/bash
  • -t: 在新容器内指定一个伪终端或终端。
  • -i: 允许你对容器内的标准输入 (STDIN) 进行交互。

退出终端，输入 exit 即可。

后台模式和端口映射：

docker run -d -p 80:80 docker/getting-started

-d：后台模式

-p：将主机的 80 端口映射到 container 的 80 端口。

后台模式会返回一大长串字符，这个长字符叫做容器 id，对于每个容器都是唯一的。

容器

docker pull xxx # 拉取镜像
docker export <id> # 导出容器
docker import # 导入镜像 

docker start <id> # 启动容器
docker run xxx xxx # 在容器内运行程序
docker stop <id> # 停止容器
docker restart # 重启容器
docker rm # 删除容器

docker attach <id> # 进入容器，退出后会自动停止运行
docker exec <id> command # 

docker ps # 查看正在运行的容器
docker port <id> # 查看容器的端口映射情况
docker logs <id> # 查看容器的日志
docker top <id> # 查看进程
docker inspect <id> # 查看 Docker 的底层信息
